# 마크다운 문법

## 1. 마크다운(Markdown)이란?
   쉽고 빠르게 문서를 만드는 문법
    > 기호 몇 개로 문서를 꾸밀 수 있는 글쓰기 방식

## 2. 마크다운의 목적(만들어진 이유)
    '사람이 읽기 쉬운 문서'를 만들기 위해 고안

## 3. 마크다운의 장/단점
### 3-1. 장점 
        - 문법이 단순하여 배우기 쉬움(개발 지식이 없더라도 사용 가능)
        - 깃허브(GitHub)와 노션(Notion) 등에서 호환이 매우 좋음
        - 문서가 깔끔하고 구조적으로 정리됨
        - 개발자들이 협업을 할 때 기본 언어로 사용됨
        - 빠른 글 작성 가능 (HTML보다 적은 문법으로 빠르게 작성)

### 3-2. 단점
        - 디자인 표현의 한계
        > '간단함'이 장점이라 복잡한 꾸미기(글자 크기 조정, 색 변경 등)가 거의 불가함
        >> 복잡한 문서 작업에는 적합하지 않음


## 4. 마크다운이 쓰이는 곳
    - 깃허브(GitHub) > 기본 문법이 마크다운
    - 노션(Notion) > 마크다운 기반 에디터(자동 변환)
    - VSCode > 개발자들의 문서 기본 형식(표준) = 마크다운
    - 개발블로그(Velog, GitHub Pages 등) > 마크다운 기반
